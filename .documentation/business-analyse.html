Introduction

Le workflow Business Analyse est un systeme optimise pour produire deux types de livrables : une documentation technico-fonctionnelle pour validation humaine, et un prompt de developpement pour implementation oneshot par Claude Code.

The Business Analyse workflow is an optimized system to produce two types of deliverables: a functional-technical documentation for human validation, and a development prompt for oneshot implementation by Claude Code.

⚠
Principe fondamental
Core Principle

Le BA est un expert technique qui comprend le code. Il produit des specifications suffisamment precises pour une implementation oneshot sans questions.

The BA is a technical expert who understands code. It produces specifications precise enough for oneshot implementation without questions.

💰
Optimise pour les tokens
Token Optimized

Agents conditionnels, sections dynamiques, options opt-in. Economie de 40-70% des tokens par rapport a une approche exhaustive.

Conditional agents, dynamic sections, opt-in options. 40-70% token savings compared to exhaustive approach.

Workflow

Le workflow s'adapte automatiquement a la complexite de la demande :

The workflow automatically adapts to the request complexity:

Track Criteres Criteria Phases Phases
QUICK ≤3 fichiers, pas de nouvelle entite, pas de nouvelle page ≤3 files, no new entity, no new page :quick → /implement
FULL Nouvelle entite, nouvelle page, module complet New entity, new page, full module :1-explore → :2-spec → :3-handoff → /implement

Architecture

Business Analyse stocke ses fichiers dans .claude/ba/ avec un sous-dossier par feature. Cette structure permet de garder l'historique des specifications et de les reutiliser pour les evolutions.

Business Analyse stores its files in .claude/ba/ with a subfolder per feature. This structure keeps specification history and allows reuse for future evolutions. 

my-project/
├── .claude/ba/                   # Business Analysis folder
│   ├── user-authentication/       # Feature: User Authentication
│   │   ├── explore.md             # Exploration results (agents output)
│   │   ├── spec.md                # Technical-functional specification
│   │   ├── handoff.md             # Implementation brief for Claude Code
│   │   └── mockup.html            # (optional) HTML mockup if --mockup
│   │
│   ├── payment-gateway/           # Feature: Payment Gateway
│   │   ├── explore.md
│   │   ├── spec.md
│   │   └── handoff.md
│   │
│   └── quick/                     # Quick fixes (no subfolder)
│       ├── add-logout-button.md   # Combined spec+handoff
│       └── fix-validation.md
│
└── src/                           # Source code (target of /implement)

Quick vs Full Track

Quick vs Full Track

For small changes (≤3 files, no new DB entity, no new page).

Combines spec + handoff in a single compact output. Launches only the explore-api agent for quick context. The file is saved to .claude/ba/quick/[feature-name].md. Ideal for: adding a button, fixing a bug, small UI modification.

/business-analyse:quick Add logout button to header

Pour les features complexes (nouvelle entite, nouvelle page, module complet).

Phase 1: Explore

/business-analyse:1-explore FULL Track

Cette commande lance les agents d'exploration en parallele pour collecter tout le contexte necessaire. Elle analyse les mots-cles de la demande pour selectionner automatiquement les agents pertinents (maximum 3 pour optimiser les tokens). L'agent explore-api est toujours inclus comme base. Les resultats sont consolides dans .claude/ba/[feature]/explore.md avec des references file:line pour chaque pattern trouve.

This command launches exploration agents in parallel to collect all necessary context. It analyzes request keywords to automatically select relevant agents (maximum 3 to optimize tokens). The explore-api agent is always included as base. Results are consolidated in .claude/ba/[feature]/explore.md with file:line references for each pattern found. 

/business-analyse:1-explore Add user management with roles and permissions

Phase 2: Specification

/business-analyse:2-spec Pour humains

Cette commande genere la documentation technico-fonctionnelle destinee a la validation humaine (PO, Tech Lead, Client). Elle lit explore.md et produit un document structure avec uniquement les sections pertinentes. Les options --mockup et --plantuml permettent d'ajouter des visuels si necessaire. Le fichier est sauvegarde dans .claude/ba/[feature]/spec.md.

This command generates functional-technical documentation for human validation (PO, Tech Lead, Client). It reads explore.md and produces a structured document with only relevant sections. The --mockup and --plantuml options add visuals if needed. The file is saved to .claude/ba/[feature]/spec.md. 

# Generate specification
/business-analyse:2-spec

# With HTML mockup (adds ~2000 tokens)
/business-analyse:2-spec --mockup

# With PlantUML ER diagram (adds ~800 tokens)
/business-analyse:2-spec --plantuml

# Both options
/business-analyse:2-spec --mockup --plantuml

Phase 3: Handoff

/business-analyse:3-handoff Pour Claude Code

Cette commande transforme la specification validee en un brief d'implementation concis pour Claude Code. Elle utilise un format diff-like avec des instructions precises : fichiers a creer/modifier, numeros de ligne, patterns a copier, et commandes a executer. Le fichier est sauvegarde dans .claude/ba/[feature]/handoff.md et peut etre passe directement a /implement.

This command transforms the validated specification into a concise implementation brief for Claude Code. It uses a diff-like format with precise instructions: files to create/modify, line numbers, patterns to copy, and commands to run. The file is saved to .claude/ba/[feature]/handoff.md and can be passed directly to /implement. 

# Generate handoff after human validation
/business-analyse:3-handoff

# Then implement with Claude Code
/implement .claude/ba/user-management/handoff.md

Exemple complet Complete Example

Voici un exemple complet du workflow Business Analyse pour une feature complexe :

Here's a complete example of the Business Analyse workflow for a complex feature:

# 1. Start with /business-analyse (auto-detects FULL track)
/business-analyse Add user management with roles and permissions

# Output:
# TRACK: FULL
# REASON: New entity (User), new page, permissions required
# AGENTS: explore-schema, explore-permissions, explore-api
# → Run /business-analyse:1-explore to start

# 2. Explore the codebase (agents run in parallel)
/business-analyse:1-explore

# → Creates .claude/ba/user-management/explore.md
# → Contains: existing patterns, file:line references, recommendations

# 3. Generate specification for human review
/business-analyse:2-spec --plantuml

# → Creates .claude/ba/user-management/spec.md
# → Contains: requirements, data model, UI spec, permissions, PlantUML diagram
# → STOP: Wait for human validation

# 4. After validation, generate implementation brief
/business-analyse:3-handoff

# → Creates .claude/ba/user-management/handoff.md
# → Contains: CREATE/MODIFY instructions, line numbers, commands

# 5. Implement with Claude Code (oneshot)
/implement .claude/ba/user-management/handoff.md

# → Claude Code reads handoff.md and implements everything in one pass
# → No questions asked - all info is in the handoff
